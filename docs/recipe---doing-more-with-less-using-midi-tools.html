6 Recipe - Doing more with less using MIDI tools

  1. Create a MIDI track but do not add an instrument to that track. This is going to be the “generator” for our track.

  2. Within this track, add MIDI clips like those created in Recipe - Changing chords. Or you can use a Max for Live device like the (free Max for Live plugin) Nordmann Chord Generator to generate chords. Name this the “Generator” track (CTRL+R to rename).

  3. Add MIDI tracks and assign the input for these tracks to be the “Generator” track that you have just created above.

  4. In your tracks, you can add MIDI plugins to alter the incoming chord. For example:

    Here I am using a Velocity plugin to change and add a little randomness to the velocity output. Some virtual instruments can use velocity to alter the timbre of the produced sound, for example changing the filter cutoff frequency.

  5. Use random note probability (a new feature in Live 11) on all notes in the chord, set at about 50-60% probability so that each time the clip triggers, the note choice / chord is different. This will mean that sometimes the chord will have 4 notes, but it could vary between 2 and 7… (in fact it could vary between none and 8, but the the extremes are very unlikely).

  6. Make sure that the clip lengths vary between tracks (all notes within the clip have the same length). So clip1 in track 1 could have 4 beat (1 bar) length; clip 1 in track 2 could have 5 beat length; clip 1 in track 3 could have 6 beat length etc. The important thing is not the clip length, but the fact that they vary across tracks.

  7. ALSO, vary clip lengths between scenes within tracks. So a track that has 4 beats in clip 1, maybe has 5 beats in clip 2 and 6 beats in clip 3. Similarly, the exact length is unimportant. The important thing is that the clip lengths vary both within tracks and between tracks. AVOID having the same clip lengths in the same scenes.

    Following the advice above you should wind up with something like this in the first three clip slots of the three tracks:

    Track 1 Track 2 Track 3
    Clip 1 - Amin - 4 beats Clip 1 - Amin - 5 beats Clip 1 - Amin - 6 beats
    Clip 2 - Dmin - 5 beats Clip 2 - Dmin - 6 beats Clip 2 - Dmin - 4 beats
    Clip 3 - Gmin - 6 beats Clip 3 - Gmin - 4 beats Clip 3 - Gmin - 5 beats

  8. Set up Follow Actions (see the following chapter Process - Follow Actions) to progress to Next clip at end of each clip with 50% probability, with the other action set to “Play again” with 50% probability. When you launch the scene, each clip will play to the end of the clip, but because individual note probability within each of the stacked chords is set to 50-60%, the chord will sound different each time it plays. Also, the randomness we’re using for note probability also means that the same chord clip in each track is also likely to sound different. The 50-50% split of next action - “Play again” or “Next” - will mean that some tracks may progress to the next chord while some will play the same clip again.

  9. Choose sounds for each track that complement each other. Could be pads, arpeggios, textures etc. Arpeggios and sounds that evolve are particularly useful in this recipe. You can automate parameters within clips as well to introduce additional movement in the sound. An important point is that the sound shouldn’t be too static.

  10. Here I’m suggesting using (relatively) short clip lengths in order to quickly demonstrate how the recipe works. The recipe will also work with longer clips lengths - for example substituting bars for beats above. Having longer clips means that the playing clip has longer for the sound to evolve or for automation to happen (see Step 8). It’s a matter of preference, choice of sound, etc.

  11. Press play. Sit back and relax.
